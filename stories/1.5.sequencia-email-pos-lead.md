## Story

**As a** lead que preencheu o formulário de interesse (Pro/Premium),
**I want** receber emails úteis e relevantes nos próximos 10 dias,
**so that** eu tenha informação suficiente para decidir agendar a call de qualificação.

## Acceptance Criteria

1. 5 emails em cadência automática: dia 0 (imediato), dia 2, dia 5, dia 7, dia 10 — conforme copy kit
2. Enviados via Resend com personalização por nome do lead (`{{nome}}`)
3. Campo `email_sequence_step` na tabela `consulting_leads` rastreia qual email foi enviado por último (0 = nenhum, 1–5 = emails enviados)
4. Idempotência: se o job rodar duas vezes para o mesmo lead no mesmo step, nenhum email duplicado é enviado
5. Link de unsubscribe funcional em cada email — ao clicar, `email_sequence_step` é atualizado para `-1` (opted out) e nenhum email adicional é enviado
6. Cron job ou Supabase Edge Function que processa a fila de emails a cada hora
7. E-mail 1 (dia 0) já é disparado na Story 1.3 — esta story implementa os E-mails 2-5 e o mecanismo de agendamento

### HMAC para Token de Unsubscribe

```typescript
// lib/email/unsubscribe.ts
import { createHmac } from 'crypto';

const SECRET = process.env.UNSUBSCRIBE_SECRET!;

export function generateUnsubscribeToken(leadId: string): string {
  const payload = `${leadId}:unsub`;
  const hmac = createHmac('sha256', SECRET).update(payload).digest('hex');
  return Buffer.from(`${leadId}:${hmac}`).toString('base64url');
}

export function validateUnsubscribeToken(token: string): string | null {
  try {
    const decoded = Buffer.from(token, 'base64url').toString('utf-8');
    const [leadId, providedHmac] = decoded.split(':');
    const expectedHmac = createHmac('sha256', SECRET).update(`${leadId}:unsub`).digest('hex');
    if (providedHmac !== expectedHmac) return null;
    return leadId;
  } catch {
    return null;
  }
}
```

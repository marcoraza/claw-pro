## Story

**As a** empresário decidido a contratar o tier Essencial,
**I want** pagar R$1.500 diretamente na página sem precisar entrar em contato,
**so that** minha sessão seja agendada e eu receba as próximas instruções imediatamente.

## Acceptance Criteria

1. Página `/consultoria/checkout` com checkout via Mercado Pago (Pix + cartão de crédito) para R$1.500
2. Após confirmação de pagamento, redireciona para `/consultoria/confirmacao`
3. Página `/consultoria/confirmacao` exibe: instruções claras sobre o formulário de discovery, link direto para o formulário (Story 1.4), prazo (48h antes da sessão)
4. Registro criado na tabela `consulting_leads` com `status = 'converted'`, `tier_interest = 'essencial'` e `payment_id` preenchido
5. Email de confirmação enviado via Resend com: subject, instruções do formulário, link do formulário e prazo
6. UTM params capturados da URL de origem e salvos no `consulting_lead` (utm_source, utm_medium, utm_campaign)
7. Página de confirmação funciona corretamente mesmo se o usuário recarregar (idempotente)

### Tabela consulting_leads

```sql
CREATE TABLE consulting_leads (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  name TEXT NOT NULL,
  email TEXT NOT NULL,
  business TEXT,
  urgency TEXT CHECK (urgency IN ('low', 'medium', 'high')),
  tier_interest TEXT CHECK (tier_interest IN ('essencial', 'pro', 'premium')),
  status TEXT CHECK (status IN ('new', 'contacted', 'qualified', 'converted', 'lost')) DEFAULT 'new',
  discovery_form JSONB,
  session_date TIMESTAMPTZ,
  payment_id TEXT,
  utm_source TEXT,
  utm_medium TEXT,
  utm_campaign TEXT,
  referral_code TEXT,
  email_sequence_step INT DEFAULT 0,
  created_at TIMESTAMPTZ DEFAULT now(),
  updated_at TIMESTAMPTZ DEFAULT now()
);
```
